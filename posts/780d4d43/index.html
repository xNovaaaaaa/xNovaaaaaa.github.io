与地面的碰撞判断-OverlapCapsule

在之前做了一个跳跃的状态，但是要让跳跃完美在落地的时候结束，就要判断何时与地面发生碰撞

在这里使用了Physical.OverlapCapsule这个方法

该方法是给定一个胶囊体，检查其与哪些物体的Collider发生碰撞，返回这些Collider

public static Collider[] OverlapCapsule (Vector3 point0, Vector3 point1, float radius, int layerMask= AllLayers, QueryTriggerInteraction queryTriggerInteraction= QueryTriggerInteraction.UseGlobal);

参数
point0 胶囊体在 start 处的球体中心
point1 胶囊体在 end 处的球体中心
radius 胶囊体的半径
layerMask 层遮罩，用于在投射胶囊体时有选择地忽略碰撞体
queryTriggerInteraction 指定该查询是否应该命中触发器

详细介绍和更多相关的方法可以看小功能丨关于Unity Collider Physics.Overlap

看源码

核心思想是实现感应器位置的实时更新，随后判断是否碰撞并返回碰撞体

如果碰撞体的数量大于0说明发生了碰撞，否则没发生碰撞

需要注意的是，方法的layerMask参数只接收状态层的mask，但是我们只知道状态层的名字

所以要用LayerMask.GetMask(string layerName)方法来获取将名字来转换成mask

要实现像黑魂那样一个键短按一下是翻滚，长按是奔跑这样的功能

我们需要知道按了多长时间才算是奔跑信号，按了不到那么长的时间就是翻滚信号

所以需要一个计时器，所以首先写一个计时器类MyTimer

这个类没有extendsMonoBehaviour不会随着游戏进行更新，所以写一个Tick方法让这个类能在ActorController里调用

用一个MyButton类来处理这三种信号

这个类也有一个Tick方法来保证能随着游戏更新

其他两个的很好理解，要解释“一键两用”这种信号，先借用一下傅老师的图

红色线是普通的按压信号波形

紫色是按压“一键两用”这个键的信号

黄色线是“一键两用”中奔跑信号

则黄色渐变区域（delay区域）就是翻滚信号的输入时间段，在紫色上升（玩家按压）之后，黄色上升之前松手，就是翻滚；那如果一直按压超过了delay区域的时间，黄色上升了，进入奔跑状态

那斜线区域（extend区域）用于双击信号，这里我没用到，傅老师用来实现黑魂1中的跳跃操作方式，那个有点反人类，我使用了艾尔登法环中的跳跃操作方式

怎么控制这两个时间区域的大小呢，就用前面写的MyTimer类了
