Performance Implications of Recursion

What happens to the stack in a recursion method?

  • On stack frame for each invocation

  • One copy of each method’s instance variables on the stack

Recursive methods in Java generally:
  • Take larger RAM footprint than iterative
  • Footprint grows linearly with number of invocation (stack explosion)

Classification of Recursion

  • Head Recursion
  • Middle Recursion
  • Tail Recursion
