Performance Implications of Recursion

What happens to the stack in a recursion method?

  • On stack frame for each invocation

  • One copy of each method’s instance variables on the stack

Recursive methods in Java generally:
  • Take larger RAM footprint than iterative
  • Footprint grows linearly with number of invocation (stack explosion)

Classification of Recursion

  • Head Recursion
  • Middle Recursion
  • Tail Recursion

Interators

We often need to iterate over a collection either to find or process data

  • An instance of Iterator is returned when iterator() is called on a collection
  • Use an iterator to traverse a collection
  • Unlike a for-each loop, an iterator allows modifications to a collection

Access Control Modifiers

Access Control Modifiers for Classes

  • public: A class that forms part of an exposed class library
  • no modifier: A (utility) class that internally supports a class library
  • protected: There are no protected classes
  • private: Singleton pattern or inner classes to support elegant implementation of an outer class

Suggestion for Access Control Modifiers

Always start off by declaring private variables and methods. Only change the access level if it is absolutely necessary

Final Modifier for Immutability

  • Final classes cannot be extended
  • Final methods cannot be changed/overridden
  • Final variable prevents further write operations

Static Modifier

Class Variable

  • Accessible without creating a class instance
  • useful for data unique to a class, not an instance

Class Method

  • Static methods operate on classes and do not require classes to be instantiated
  • Static methods cannot access any instance variables
  • Useful if a more procedural programming is needed!

7 Erlang

Introduction to Erlang

Why Erlang

  • To develop software quickly and efficiently
  • To develop systems that are tolerant of software errors and hardware failures
  • To update software on the fly

Features of Erlang

  1. High-Level Constructs
  2. Concurrent Processes and Message Passing
  3. Scalable, Safe and Efficient Concurrency
  4. Soft Real-Time Properties
  5. Robustness
  6. Distributed Computing Capabilities
  7. Integration and Openness

High-Level Constructs

  • Erlang is a declarative language
  • Declarative languages describe what a program should accomplish rather than how to accomplish it

Concurrent Processes and Message Passing

  • Rather than providing threads that share memory, each Erlang process executes in its own memory space and has its own heap and stack

  • Processes cannot interfere with each other inadvertently leading to deadlocks and other horrors

  • Processes communicate with each other via message passing, where the message can be any Erlang data value

  • Message passing is asynchronous (i.e., once a message is sent, the process can continue processing)

Basic Erlang

Integers

  • Integet (Decimal)
  • BigNums
    • Can do operations with numbers of an arbitrary size
    • Do not overflow unless you run out of memory
  • Although decimal is assumed by default, integers can specify a number in any base between 2 and 16
    • 2#1010 (binary)
    • 16#EA (hexadecimal·)
