2 让角色平滑移动

第一节中的代码可以让角色移动和，但是速度都是突变到1的，这样会让角色的移动变得十分生硬，为了让角色能够更加平滑地移动，我们可以使用Mathf中的SmoothDamp函数

首先看一下函数语法

public static final SmoothDamp(float current, float target, ref float currentVelocity, float smoothTime, float maxSpeed = Mathf.Infinity, float deltaTime = Time.deltaTime);

一般只需要下方四个参数

参数 啥意思
current 当前数值
target 目标数值
currentVelocity 当前速度，不需要关心，调用函数时这个值由函数控制
smoothTime 从当前数值变化到目标数值所需要的时间

4 新增动画控制器

在项目目录下新建一个文件夹命名为Animator，这个文件夹将被用来存放动画控制器(Animator)

然后右键文件夹创建一个Animator Controller，将控制器命名为Actor

随后在角色模型ybot的Inspector中将Controller设置为Actor，这样就可以通过Actor来控制ybot的动作

双击动画控制器Actor可以打开Animator，将动作比如Idle或者Walk就可以进行调整，右键Animator中的动作然后选择Set as Layer Default State可以在运行时就循环运行该动作

但是只是如此的话我们会发现动作之间进行转换时，比如从Idle转换为Walk，动作会非常生硬，像是从一个动作直接转变成另外一个动作，这个问题我们将在下一节解决

5 一维混合树

混合树的作用简单来说就是混合多个动作，通过一个变量来控制动作之间连续而非离散地转换。这样就会让动作之间的转换变得连贯，解决了上一节转换生硬的问题

我们开始，首先删除Animator窗口中的Idle动作，然后右键空白区域选择Create State-From New Blend Tree，就可以创建一个空白的混合树

点击该混合树在Inspector中将其重命名为Ground，意思是该树保存角色在地面上做的动作

随后右键该树将其设为默认状态

双击混合树Ground进入混合树单独的页面，在Inspector页面中可以将动作添加进混合树，点击加号-Add Motion Field

这里我们添加两个动作，一个是Idle，一个是Walk，将动作拖入Motion List里的None(Motion)区域即可添加成功。Threshold控制动作在什么时候完全做出，可以在上方蓝色框图中调整Threshold，也可以拖动红色播放条来观察混合后的动作

前面我们提到可以用一个参数来控制动作的切换，这个参数在Animator中可以看到，我们可以看到添加的两个动作也已经显示在混合树里了，这里也需要再给混合树改个名字。本来这里的Blend就是我们所说的控制动作的参数，我们双击左边的变量命把它改成forward，意思是我们用前进来控制该混合树。拖动forward右边的滑动条可以控制forward的值，也可以看到动作做相应变化

下节将学习怎么用使用参数来连续地控制动作
